## What Fluxurl is

A URL shortener with click analytics. Solo learning project, not a real product.

## Why this project exists

This is a **learning project**, not a shipping project. The goal is depth of understanding, not speed of delivery. Specifically:

- Learn Python and FastAPI (no prior Python experience)
- Learn Docker properly (not just copy a Dockerfile)
- Learn AWS — specifically EC2, ECR, IAM
- Learn CI/CD end-to-end with GitHub Actions
- Learn HLD/LLD thinking — design decisions I can defend in interviews

The infrastructure and pipeline are the star. The app is the vehicle.

## How I want help

- **Explain, don't just generate.** Code without a "why" doesn't help me. If you write code, walk me through what it does and the alternatives I'm not picking.
- **Don't skip ahead.** If I'm in Phase 2, don't pre-emptively add Phase 5 patterns "for later." Each phase is sequenced deliberately.
- **Push back when I'm wrong.** I'd rather be corrected than agreed with. If I suggest something that conflicts with the learning goal, say so.
- **Suggest things to break intentionally.** Watching things fail is a core part of the learning approach here.
- **Don't over-engineer.** If a simpler approach teaches the same lesson, prefer it.

## Constraints

- AWS free tier only — zero spend
- Solo developer, no deadline
- Manual first, automate second — never automate something I haven't done by hand
- Every meaningful decision gets a `NOTES.md` entry in my own words

## Stack (v1)

- **App:** Python 3.12, FastAPI, Pydantic, SQLAlchemy 2.0 async, Alembic, Uvicorn
- **Testing:** pytest, ruff
- **Database:** PostgreSQL (Docker container on EC2 in v1; RDS later)
- **Packaging:** Docker (multi-stage), docker-compose for local dev
- **AWS:** EC2, ECR, IAM (instance role), VPC + Security Groups, CloudWatch
- **CI/CD:** GitHub Actions

## Out of scope for v1 (do not suggest)

- Kubernetes, ECS, Fargate, Lambda
- CloudFront, Route 53, API Gateway
- Redis, Celery, separate frontend
- Poetry/Pipenv (using `pip` + `requirements.txt`)
- Multiple environments (single env is fine)

These are *deliberately* deferred to later phases or other projects.

## Conventions

- Async everywhere — `async def`, async SQLAlchemy session, async tests
- Type hints on every function signature
- Pydantic for any data crossing an API boundary
- Errors return proper HTTP status codes, not 200 with `{"error": ...}`
- One concern per module — don't bundle unrelated logic
